gkd ai improved 项目，使用 ai增强了部分功能。

> GKD-Aim 是基于 GKD 的非官方修改版，不属于 gkd-kit 官方项目。使用问题、功能建议和崩溃反馈请提交到 [GKD-Aim Issues](https://github.com/AngryDisk/GKD-Aim/issues)，请勿向 GKD 官方项目反馈本修改版的问题。此标识与链接调整遵循 [gkd-kit 关于第三方修改版的说明](https://github.com/orgs/gkd-kit/discussions/679)。

## 项目来源

经常使用gkd的朋友们可能知道，在gkd安装之后，一般要找一个成品规则订阅。这个订阅质量好坏，直接会影响到gkd日常的使用体验。

这个订阅规则做的再好，偶尔也会碰到一些无法跳过的广告。可能是订阅中的配置没有开、也可能是因为没有其他人碰到过类似的广告。

但是自己手动添加规则又不会；花时间学的话又不值当，又找不到几个广告。

为什么不把这个规则生成的过程交给ai呢？

这就是这个项目做的事情。

## 相比gkd新增的功能

- 在高级设置中，增加了2个配置，一个是 deepseek的 api key；一个是 AI规则生成
- 开启以上配置，在生成快照的时候，直接就会请求AI，让deepseek AI生成一个规则
- 如果自动的规则生成不好用，也可以去快照列表，可以对快照进行补充描述，让ai生成规则

以上功能绕过了复杂的规则调整配置，直接把这种烦人的劳动甩给ai，极大简化了写规则的流程。甚至能让gkd不使用订阅，碰到一个广告就能解决一个广告。

## FAQ

### ai能别写所有的广告规则么？

答：不能，有时候AI会没有响应，有时候AI就是没有结果（手动写规则，也写不出来的）

### 目前只支持deepseek么？

答：是的，因为手里有API key，随便写的

### 项目是AI写的么？

答：是的，codex干的，我什么都没干，就写了个readme

### 如何手动添加推则？

答：我大概知道个流程，没实际操作过。

1. 使用gkd内置的快照功能，记录当前页面的结构和截图
2. 如果有必要的话，需要对这个快照进行脱敏，去掉快照中的隐私信息
3. 使用gkd提供的审查工具，把以上快照导入
4. 在审查工具中，找到可以跳过广告的点击内容，生成一个类似于 xml筛选的语法
5. 把这个筛选粘贴到 gkd的规则中，或者是配置中

### 为什么gkd不能搞定所有广告？

答：据我目前观察，gkd没有做ocr相关的功能，只是根据当前程序的页面结构，进行广告跳过。如果广告和正常UI混杂在一起，没有OCR就没办法辨认出 是广告还是正常UI。

换句话说，就算是人工去写广告跳过的规则，能力也是有限的。

### 会长期维护么？

答：不会，我写出这个项目，一个目的是为了试一下AI，另一个是为了把这个IDEA公开出来。长期维护不是我想要的，而且项目配套的很多网站功能，我都提供不了，也不想提供。

最好的结果是被官方合并；次好的结果是有人把idea抄走，并长期维护；最次的结果就是抄过去的人，让我继续更新。

多么实用好用的功能啊，赶快来抄吧！
