# DeepSeek AI 广告规则

此分支在 GKD 现有的快照按钮和本地订阅之上增加了一个可选的 AI 规则生成流程，没有改变远程订阅格式或规则引擎。

## 使用方法

1. 打开 `设置 → 高级设置 → 快照`。
2. 点击独立的 `DeepSeek API Key` 设置项，填写并保存 API Key；保存 Key 不会自动开启 AI 模式。
3. 开启 `AI 规则生成` 和 `快照按钮`。启用 AI 模式后，悬浮按钮图标会变为自动化图标。
4. 未被订阅规则覆盖的广告出现时，点击一次悬浮按钮。
5. GKD 会保存快照、让 DeepSeek 判断关闭/跳过节点、生成并校验选择器，然后把新规则写入并启用 `本地订阅` 对应的应用中。

也可以打开 `快照记录`，点击一条已经保存的快照，选择 `补充描述+ai规则生成`，输入关闭按钮的位置、文字或外观后直接生成规则。此流程复用该条快照，不会重新抓屏。

未填写 DeepSeek API Key 时，`AI 规则生成` 开关和快照记录中的描述生成入口均不可用；清除 Key 会自动关闭 AI 模式。

生成成功后，规则可在 `订阅 → 本地订阅 → 应用规则` 中查看、编辑或删除。AI 不能可靠判断目标节点或无法生成唯一选择器时，只保留快照，不写入规则。

## 安全与隐私

- DeepSeek 只负责从候选节点中选择应点击的节点，不直接生成可执行的 GKD 选择器。
- GKD 使用本地确定性代码生成选择器，并要求它在快照上只命中 AI 选中的单个节点；随后还会用现有规则解析器校验。
- 请求会发送当前应用 ID、Activity ID，以及最多 450 个可见无障碍节点的文本、描述、控件 ID、位置等信息。使用快照记录入口时，还会发送用户填写的补充描述（最多 500 个字符）。
- 截图不会发送给 DeepSeek，但无障碍节点中的文字仍可能包含隐私信息。
- API Key 存放在现有 `private-store` 中，不包含在 GKD 的普通备份和诊断日志导出内。
- 当前使用 DeepSeek 官方 `https://api.deepseek.com/chat/completions` 接口和 `deepseek-v4-flash` 模型。
- 请求会显式关闭 V4 默认的思考模式，避免短 JSON 答案的输出额度先被思考内容耗尽；服务端偶发返回空内容时会自动重试一次。

## 已知限制

- 只有截图、没有可识别无障碍节点的广告无法生成规则。
- 没有文字或控件 ID 的关闭图标，只有在节点关系或右上角位置足以形成唯一选择器时才会生成。
- 规则默认限制为当前 Activity，并在每次进入应用后最多触发一次；这是为了降低误点风险，可在本地订阅中手动调整。
